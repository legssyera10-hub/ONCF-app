# Documentation Projet

Ce dossier centralise la documentation du projet pour garder une racine propre et lisible sur GitHub.

- [QUICKSTART.md](QUICKSTART.md): lancement rapide local et Docker.

- [assets/processus_acheminement_materiel_roulant.png](assets/processus_acheminement_materiel_roulant.png): schema de processus.

## Convention de roles

Pour la documentation fonctionnelle, la plateforme utilise 4 roles metier:

1. `Technicentre` (demandeur / recepteur).
2. `Permanent PM`.
3. `Admin`.
4. `Suivi`.

Certaines references techniques historiques peuvent encore apparaitre dans le code pour compatibilite.
